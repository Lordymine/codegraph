We delegate to the type checkers you already run.

You don't need to re-implement a type checker to build an accurate code knowledge graph for LLMs.

Call resolution is handed to scip-typescript for TS/JS and go/packages with a VTA call graph for Go. That buys parameter binding, overload inference, interface-dispatch narrowing, and same-name disambiguation for free, plus an index roughly 36 times faster than the embedding-based prior art.

Honest precision

An edge whose endpoints aren't both real nodes is dropped, never guessed. A missing edge beats a wrong one, because the agent must never be sent the wrong way.

Compact by construction

Every query returns one tab-separated line, never source. Code arrives only when you ask for a snippet. That selectivity is where the tokens are saved.

Incremental and instant

Re-index is a no-op when nothing changed, and the MCP server indexes in the background, so the agent always opens a fresh graph with no manual step.

Nine tools your agent can call.

Every one returns compact references, except snippet, which returns source only when you actually need code.

get_architectureA one-shot repo map: languages, packages, and hotspots, to orient before diving in.
callersEvery function that calls a symbol, type-checker resolved, decoys removed.
calleesWhat a symbol calls, intra-repo. The other half of the call graph.
searchRanked BM25 symbol search returning compact references, not source.
similarNear-clones via MinHash and LSH, to surface copy-paste worth refactoring.
dead_codeUncalled private functions: candidates to investigate, honestly bounded.
neighborsEverything connected to a symbol, in both directions.
snippetThe one tool that returns source, for when you do need to read code.
detect_changesWhat changed since the last index, so the graph is never silently stale.
